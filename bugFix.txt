# Рабочий проект 3-го семинара

## Настройка совместной работы

Для начала работ необходимо создать аккаунт на GitHub.com и создать локальный репозиторий используя следующие программы

- **git clone**
Команда **git clone** позволяет копировать внешний репозиторий на свой ПК. Данная команда составная: загружает все изменения
и сливает (merge) все ветки на локальном компьютере и в удаленном репозитории.

- **git pull**
Эта команда позволяет скачать все из текущего внешнего репозитория и автоматически сделать merge с нашей версией.

- **git push**
Для отправки своей версии локального репозитория во внешний репозиторий в GitHub необходимо выполнить команду <git push>. 
При первом ее использовании требуется авторизация.

### Настройка совместного проекта

Для добавления нового удаленного репозитория на GitHub из локального репозитория, в командной строке необходимо прописать:

- **git remote add origin <адрес репозитория>** - создание репозитория по указанному адресу
- **git branch -M main** - создание ветки main (необязательная процедура)
- **git push -u origin main** - отправка репозитория в GitHub

### Выполнение pull request (запрос на выкачивание)

* Делаем **fork** (ответвление) репозитория
* Делаем _git clone_ **СВОЕЙ** версии репозитория
* Создаем новую ветку и в нее в носим свои изменения
* Фиксируем изменения (делаем коммиты)
* Отправляем свою версию в свой GitHub
* На сайте GitHub нажимаем кнопку _pull request_
